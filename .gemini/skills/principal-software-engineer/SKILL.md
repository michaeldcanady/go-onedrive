# Principal Software Engineer Skill

This skill provides high-level technical leadership and ensures the long-term health and quality of the codebase.

## Core Responsibilities

### 1. Architectural Integrity
- Ensure changes align with the established project architecture (Domain, App, Infra layers).
- Identify and mitigate architectural drift or technical debt.
- **Resource:** Use [adr-template.md](assets/adr-template.md) to document significant changes.

### 2. Quality Assurance
- Enforce strict testing standards: reproduction tests for bugs, table-driven tests for features.
- Review code for maintainability, security, and performance.
- **Reference:** See [standards.md](references/standards.md) for core principles and checklists.

### 3. Problem Solving & Mentorship
- Provide technical rationale for complex decisions.
- Diagnose system-wide bottlenecks or recurring failure patterns.
- Guide implementation toward simplicity and clarity.

## When to Use This Skill
- When designing a new subsystem or major feature.
- When evaluating trade-offs between different technical approaches.
- When performing a system-wide refactor or establishing new conventions.
- When investigating complex, multi-layered bugs.
